Platform Architecture — GEZMA One Super App

Ekosistem terdiri dari 4 sub-platform yang saling terintegrasi:

GEZMA Agent (B2B)

Dashboard SaaS untuk travel agent/PPIU. CRM jamaah, manajemen paket & trip, pembayaran, laporan, AI assistant, gamifikasi, white-label branding. ~100% core product.

GEZMA Pilgrim (B2C)

Portal jamaah — 14+ halaman: tracking perjalanan, manasik online, doa & dzikir, dokumen, pembayaran, roommate matching, packing list, referral, profil.

GEZMA Academy (LMS)

Learning Management System — 12 courses, 36 lessons, quiz interaktif, sertifikat, progress tracking, course reviews.

GEZMA Command Center (Admin)

Dashboard asosiasi/regulator — compliance scoring, monitoring PPIU, auto-suspend, audit log, verifikasi dokumen.

Security Measures

  • Authentication: JWT httpOnly cookie + bcrypt password hashing + TOTP 2FA
  • Authorization: Role-based (owner/admin/staff/marketing) + granular permission per-user
  • Multi-tenant Isolation: Semua query difilter by agencyId, data antar agency terpisah total
  • Input Validation: Zod schema di setiap API endpoint
  • Rate Limiting: Login & sensitive endpoints dilindungi rate limiter
  • CSP Headers: Content Security Policy aktif
  • Environment Validation: Zod schema untuk semua env vars saat startup
  • Activity Logging: Semua create/update/delete tercatat di audit log
  • Session 16 QA: 15 bug/security fixes — termasuk race condition, missing permission checks

Code Quality

  • TypeScript strict mode — 0 errors
  • ESLint + lint-staged + pre-commit hooks
  • Centralized error handling & structured logging (JSON in prod)
  • Consistent API pattern across 138 endpoints
  • 45 dependencies only (minimal, no bloat)
